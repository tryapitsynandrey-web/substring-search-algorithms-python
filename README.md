# Порівняння алгоритмів пошуку підрядка

Було порівняно три алгоритми пошуку підрядка:

- Боєра-Мура
- Кнута-Морріса-Пратта
- Рабіна-Карпа

Для тестування використано файли `1.txt` та `2.txt`. Для кожного файлу перевірено два підрядки: один наявний у тексті та один вигаданий. Час вимірювався за допомогою `timeit`: кожен випадок запускався 1000 разів, після чого обчислювався середній час одного запуску.

## Використані підрядки

| Файл | Тип підрядка | Підрядок |
| --- | --- | --- |
| `1.txt` | Наявний | `алгоритми та структури даних` |
| `1.txt` | Вигаданий | `вигаданий підрядок якого точно немає у першому тексті` |
| `2.txt` | Наявний | `рекомендаційної системи` |
| `2.txt` | Вигаданий | `вигаданий підрядок якого точно немає у другому тексті` |

## Результати для `1.txt`

| Підрядок | Алгоритм | Час, сек | Індекс |
| --- | --- | ---: | ---: |
| наявний | Боєра-Мура | 0.010992 | 468 |
| наявний | Кнута-Морріса-Пратта | 0.045267 | 468 |
| наявний | Рабіна-Карпа | 0.081833 | 468 |
| вигаданий | Боєра-Мура | 0.150992 | -1 |
| вигаданий | Кнута-Морріса-Пратта | 1.080278 | -1 |
| вигаданий | Рабіна-Карпа | 2.047412 | -1 |

Найшвидшим алгоритмом для `1.txt` став алгоритм Боєра-Мура. Він був найшвидшим і для наявного, і для відсутнього підрядка.

## Загальний висновок

За сумарним часом виконання для двох текстів і двох типів підрядків результати такі:

| Алгоритм | Сумарний час, сек |
| --- | ---: |
| Боєра-Мура | 0.386748 |
| Кнута-Морріса-Пратта | 2.751144 |
| Рабіна-Карпа | 5.101217 |

У межах цього тесту найефективнішим загалом є алгоритм Боєра-Мура. Алгоритм Кнута-Морріса-Пратта був другим за швидкістю, а Рабіна-Карпа показав найбільший час виконання. Різниця особливо помітна під час пошуку вигаданого підрядка, якого немає в тексті.

Алгоритм Боєра-Мура демонструє високу продуктивність завдяки використанню правила поганого символу, що дозволяє пропускати частини тексту під час пошуку. Алгоритм Кнута-Морріса-Пратта працює стабільно та забезпечує лінійну складність пошуку, але в цьому тесті поступився за швидкістю. Алгоритм Рабіна-Карпа використовує хешування для порівняння підрядків, однак через додаткові перевірки збігів показав найгірший результат серед трьох алгоритмів.
