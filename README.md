# Порівняння алгоритмів пошуку підрядка

Було порівняно три алгоритми пошуку підрядка:

- Боєра-Мура
- Кнута-Морріса-Пратта
- Рабіна-Карпа

Для тестування використано файли `1.txt` та `2.txt`. Для кожного файлу перевірено два підрядки: один наявний у тексті та один вигаданий. Час вимірювався за допомогою `timeit`, по 1000 запусків для кожного випадку.

## Використані підрядки

| Файл | Тип підрядка | Підрядок |
|---|---|---|
| `1.txt` | Наявний | `алгоритми та структури даних` |
| `1.txt` | Вигаданий | `вигаданий підрядок якого точно немає у першому тексті` |
| `2.txt` | Наявний | `рекомендаційної системи` |
| `2.txt` | Вигаданий | `вигаданий підрядок якого точно немає у другому тексті` |

## Результати для `1.txt`

| Підрядок | Алгоритм | Час, сек | Індекс |
|---|---|---:|---:|
| наявний | Боєра-Мура | 0.009080 | 468 |
| наявний | Кнута-Морріса-Пратта | 0.044464 | 468 |
| наявний | Рабіна-Карпа | 0.080651 | 468 |
| вигаданий | Боєра-Мура | 0.109044 | -1 |
| вигаданий | Кнута-Морріса-Пратта | 1.068901 | -1 |
| вигаданий | Рабіна-Карпа | 1.960651 | -1 |

Найшвидшим алгоритмом для `1.txt` став алгоритм Боєра-Мура. Він був найшвидшим і для наявного, і для відсутнього підрядка.

## Результати для `2.txt`

## Загальний висновок

За сумарним часом виконання для двох текстів і двох типів підрядків результати такі:

| Алгоритм | Сумарний час, сек |
|---|---:|
| Боєра-Мура | 0.281027 |
| Кнута-Морріса-Пратта | 2.713073 |
| Рабіна-Карпа | 4.980692 |

У межах цього тесту найефективнішим загалом є алгоритм Боєра-Мура. Алгоритм Кнута-Морріса-Пратта був другим за швидкістю, а Рабіна-Карпа показав найбільший час виконання. Різниця особливо помітна під час пошуку вигаданого підрядка, якого немає в тексті.
